## Running tally

- **Runs:** 9 — across 2 repos (opn-mcp ×4, methodology ×5); modes: 3 code review, 6 design decision; 1 retrospective + 8 pre-merge.
- **Verdicts:** 6 BLOCK, 1 PUSH BACK, 2 NO STRONG OBJECTION.
- **Independent defect-catches in BOTH directions** (author-steered rounds 3/5/6 set aside):
  - **Gemini-adversary → Claude-author defects:** rounds 1, 4, **9**.
  - **Claude-adversary → Gemini-author defects:** round 8.
  A *different* model catches what the author's model misses regardless of which model authors — bidirectional support for model-agnosticism.
- **First false positive (round 9): the adversary is not infallible.** Gemini's "stale §11 references" finding was wrong (§12 was appended, not renumbered). Useful calibration — 1 false positive in ~30+ findings across 9 runs, and it was a *verifiable* factual claim the author could refute with a grep, not a judgment call.
- **The methodology resisted expansion in both directions.** PR #22 (Gemini's 6 practices) and PR #24 (Claude's counter) were *both* BLOCKed and withdrawn — each attempt re-introduced duplication / mandate / scale-to-work problems the methodology's own rules forbid. Strong evidence the document was already about as complete as it should be; "status quo" was the right answer.
- **New named failure mode: adversary-authored-replacement bias.** The author (Claude) BLOCKed #22 then authored #24, which committed milder versions of the same sins. Recorded against future "the fix is my own counter-proposal" moves.
- **Still open:** no *third* model yet (Claude↔Gemini only); small same-week sample; varied adversary tier. Author-steering confound stands for rounds 3/5/6.
