## Running tally

- **Runs:** 7 — across 2 repos (opn-mcp ×4, methodology ×3); modes: 3 code review, 4 design decision; 1 retrospective + 6 pre-merge.
- **Verdicts:** 4 BLOCK, 1 PUSH BACK, 2 NO STRONG OBJECTION. Every finding pointed at a real issue; 0 clear false positives across all seven runs.
- **The honest confound — author-steering.** Rounds 3, 5, and 6 were on the author's *own* methodology ADRs, and in each the author pre-pointed the adversary at the suspected weaknesses in the bundle. Those are **confirmations, not independent discoveries**. The genuinely *independent* catches — defects the author did not know about — are **rounds 1 and 4** (opn-mcp): the stale templates and the dropped settings. So the core hypothesis ("a different model catches what the author misses") is supported, but by 2 clean runs, not 7.
- **Model-agnosticism — first reverse data point (round 7), partial.** Roles were swapped (Gemini author, Claude adversary), un-steered. Result: the adversary did a genuine review and returned NO STRONG OBJECTION on correct work — so it does **not false-positive in reverse**. But it does **not** yet show reverse-direction *defect-catching*, because the authored change had no catchable defect. Still untested: an adversary catching a *real flaw* in a Gemini-authored change; and a third model.
- **Systemic lesson the loop taught:** drafting ran ahead of evidence — three straight design ADRs (graduation, rollout) were BLOCKed for claiming/building ahead of validation. Validate cheaply *before* designing infrastructure.
- **Next:** either (a) a reversed-role run where the Gemini-authored change *does* contain a real flaw (to test reverse defect-catching), and/or (b) a third-model run — then redesign the rollout (expand-contract migration, single shared workflow, clean supersession) and, after it lands, re-attempt graduation. Record model + tier per row.
