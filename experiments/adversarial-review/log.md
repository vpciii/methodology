## Running tally

- **Runs:** 8 — across 2 repos (opn-mcp ×4, methodology ×4); modes: 3 code review, 5 design decision; 1 retrospective + 7 pre-merge.
- **Verdicts:** 5 BLOCK, 1 PUSH BACK, 2 NO STRONG OBJECTION. Every finding pointed at a real issue; 0 clear false positives across all eight runs.
- **Independent defect-catches now exist in BOTH directions** — this is the core evidence, with the author-steered rounds (3/5/6) set aside:
  - **Gemini-adversary → Claude-author defects:** rounds 1, 4 (stale templates; dropped settings).
  - **Claude-adversary → Gemini-author defects:** round 8 (the 6-practice expansion's duplication / exclusion-contradiction / self-inconsistency).
  So a *different* model catches what the author's model misses regardless of which model authors — the model-agnosticism premise now has genuine **bidirectional** support, not just one-direction confirmations.
- **Reverse direction is now fully characterized (rounds 7 + 8):** 7 showed the reversed adversary does **not** false-positive on clean work; 8 showed it **does** catch real defects in flawed work. Together that is the honest validation the rollout/graduation were waiting on.
- **Still open:** a *third* model has never been tried (evidence is Claude↔Gemini only); sample is small and same-week; adversary tier has varied. The author-steering confound stands for the design-ADR rounds.
- **Systemic lesson the loop taught:** drafting ran ahead of evidence — three straight design ADRs (graduation, rollout) were BLOCKed for claiming/building ahead of validation. Validate cheaply *before* designing infrastructure.
- **Next:** with bidirectional evidence in hand, the rollout can be redesigned (expand-contract migration, single shared workflow, clean supersession) and, after it lands, graduation re-attempted — noting the residual third-model gap. Record model + tier per row.
